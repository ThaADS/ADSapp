# Claude Code Setup voor ADSapp

Deze configuratie document beschrijft de geïnstalleerde frameworks en hoe ze te gebruiken voor dit project.

## Geïnstalleerde Frameworks

### 1. Get Shit Done (GSD)
**Doel**: Spec-driven development met fase-gebaseerde workflow

**Gebruik voor ADSapp**:
```bash
# Start een nieuwe feature
/gsd:discuss-phase "Multi-tenant analytics dashboard"

# Plan de implementatie
/gsd:plan-phase

# Voer de fase uit met atomic commits
/gsd:execute-phase

# Verifieer het werk
/gsd:verify-work
```

**Workflow**:
1. Discussion → Requirements gathering
2. Planning → Structured implementation plan
3. Execution → Implementation met atomic commits
4. Verification → Quality assurance

### 2. Ralph Loop
**Doel**: Autonomous development loop tot completion

**Gebruik voor ADSapp**:
```bash
# Setup een nieuwe feature branch met Ralph
cd /path/to/feature
ralph-setup whatsapp-automation

# Of import bestaande specs
ralph-import specs/automation.md whatsapp-automation

# Start autonomous loop
cd whatsapp-automation
ralph --verbose

# Monitor in aparte terminal (optioneel)
ralph-monitor
```

**Status indicators**:
- `STATUS: IN_PROGRESS` - Werkt aan taken
- `STATUS: COMPLETE` - Alles gedaan
- `STATUS: BLOCKED` - Needs intervention
- `EXIT_SIGNAL: true` - Ready to exit

## Aanbevolen Workflow voor ADSapp Features

### Voor Nieuwe Features (Groot)
1. **GSD Discussion**: `/gsd:discuss-phase "Feature name"`
2. **GSD Planning**: `/gsd:plan-phase`
3. **Plugin Selection**: Install relevant wshobson agents
4. **GSD Execution**: `/gsd:execute-phase`
5. **Verification**: `/gsd:verify-work`

### Voor Quick Fixes
1. **GSD Quick**: `/gsd:quick "Fix description"`
2. **Direct implementation**: Met relevante plugins

### Voor Autonomous Work
1. **Ralph Setup**: `ralph-setup feature-name`
2. **Configure**: Edit PROMPT.md en @fix_plan.md
3. **Run Loop**: `ralph --verbose`
4. **Monitor**: Check status en logs

## Tips

1. **Start met GSD** voor grote features - het helpt bij requirements en planning
2. **Gebruik Ralph** voor langlopende, goed-gedefinieerde taken
3. **Install plugins** alleen wanneer je ze nodig hebt - houdt context schoon
4. **Combineer frameworks**: GSD voor planning → Ralph voor execution
5. **Monitor Ralph loops**: Check logs regelmatig om stuck loops te voorkomen

## Commando Overzicht

### GSD Commands
- `/gsd:new-project` - Initialize complete project
- `/gsd:discuss-phase` - Explore requirements
- `/gsd:plan-phase` - Create implementation plan
- `/gsd:execute-phase` - Execute with atomic commits
- `/gsd:verify-work` - Verify completion
- `/gsd:quick` - Fast execution for small tasks
- `/gsd:progress` - Check current status

### Ralph Commands
- `ralph-setup <name>` - Create new Ralph project
- `ralph-import <spec> <name>` - Import existing specs
- `ralph --verbose` - Start loop with logging
- `ralph --status` - Check current status
- `ralph --timeout <min>` - Set timeout
- `ralph-monitor` - Live monitoring (tmux)

### Plugin Commands
- `/plugin marketplace list` - Show all marketplaces
- `/plugin list wshobson-agents` - Show wshobson plugins
- `/plugin install <name>@wshobson-agents` - Install plugin
- `/plugin uninstall <name>@wshobson-agents` - Uninstall plugin
- `/plugin update` - Update all plugins

## Troubleshooting

**GSD not working?**
- Check `~/.claude/get-shit-done/` exists
- Verify installation: look for templates and workflows

**Ralph not in PATH?**
- Add to PATH: `export PATH="$HOME/.local/bin:$PATH"`
- Or use full path: `~/.local/bin/ralph`

**Plugins not loading?**
- Restart Claude Code session
- Check `~/.claude/plugins/marketplaces/wshobson-agents/` exists
- Verify `~/.claude/plugins/known_marketplaces.json` includes wshobson-agents

**Ralph loops not exiting?**
- Check `EXIT_SIGNAL: true` in status
- Review @fix_plan.md completion
- Look at logs in project directory
- Use `ralph --timeout 30` to limit runtime
