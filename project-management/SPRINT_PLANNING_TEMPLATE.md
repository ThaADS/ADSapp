## Sprint Planning Checklist

### Pre-Sprint Planning (Day -1)

- [ ] Review previous sprint retrospective action items
- [ ] Prepare sprint backlog from product backlog
- [ ] Estimate story points for backlog items
- [ ] Identify dependencies and risks
- [ ] Confirm team availability and capacity
- [ ] Schedule sprint planning meeting

### Sprint Planning Meeting (Day 0)

- [ ] Review sprint goals with team
- [ ] Present backlog items
- [ ] Team discusses and estimates tasks
- [ ] Commit to sprint backlog
- [ ] Identify sprint risks and mitigation
- [ ] Assign tasks to team members
- [ ] Document sprint plan

### During Sprint (Days 1-10)

- [ ] Daily standup meetings (15 min)
- [ ] Update burndown chart daily
- [ ] Track blockers and resolve quickly
- [ ] Monitor sprint progress
- [ ] Adjust scope if needed (with stakeholder approval)
- [ ] Update task statuses in tracking system

### Sprint Review (Day 10)

- [ ] Prepare demos for completed work
- [ ] Schedule sprint review meeting
- [ ] Demo to stakeholders
- [ ] Gather feedback
- [ ] Accept/reject completed stories
- [ ] Document stakeholder feedback

### Sprint Retrospective (Day 10)

- [ ] Schedule retrospective meeting (team only)
- [ ] Choose retrospective format
- [ ] Facilitate open discussion
- [ ] Identify action items
- [ ] Document insights and improvements
- [ ] Assign action item owners

### Post-Sprint (Day 11)

- [ ] Update project tracking dashboard
- [ ] Move incomplete stories to next sprint
- [ ] Update velocity metrics
- [ ] Share sprint summary with stakeholders
- [ ] Archive sprint documentation
- [ ] Plan next sprint
