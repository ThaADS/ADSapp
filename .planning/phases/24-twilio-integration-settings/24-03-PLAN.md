# Plan 24-03: Provider Selection Settings UI

## Overview
Create settings UI for switching between WhatsApp Cloud API and Twilio providers.

## Requirements Addressed
- TWWI-02: Provider switching UI

## Deliverables

### 1. WhatsApp Provider Settings Component
**File:** `src/components/settings/whatsapp-provider-settings.tsx`

Features:
- Toggle between Cloud API and Twilio
- Show connection status for each provider
- Health indicator for active provider
- Fallback configuration options
- Clear migration guidance

### 2. UI Elements

```tsx
// Provider card showing:
- Provider name and logo
- Connection status (connected/disconnected)
- Last sync time
- Health status
- "Make Active" button

// Settings panel:
- Active provider toggle
- Fallback provider dropdown
- Template preference selector
- Connection management links
```

### 3. Integration Points

- Link to Cloud API connection settings
- Link to Twilio connection settings (Phase 21)
- Real-time health updates via polling
- Toast notifications for provider switch

## Acceptance Criteria
- [x] Provider cards show connection status
- [x] Toggle switches active provider
- [x] Fallback configuration works
- [x] Settings persist and load correctly
- [x] Health indicators update in real-time
- [x] i18n translations (EN/NL)

## Dependencies
- Plan 24-02 (API endpoints)
- Existing settings page structure
